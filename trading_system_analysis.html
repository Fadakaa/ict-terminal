Key Findings

✓ Bayesian Layer IS Learning

Win rate belief evolved from 34.4% (prior from V1 seed data, which had 39% WR in London) to 76.9% after 391 observations. The belief stabilised around trade #75 at ~77% and has remained in the 76-80% band since. This is the system's only truly adaptive component — it's correctly tracking that Claude's real-world accuracy far exceeds the mechanical V1 baseline.

✓ SL Calibration Is Adding Value

The ML widened Claude's stop-loss on 69.5% of trades (173/249), saving 53 trades from being stopped out that would have lost with Claude's original tighter SLs. Claude's average SL is 1.58 ATR; calibration pushes it to 2.37 ATR (0.78 ATR wider). This is the system's most tangible value-add — it preserves winning trades that Claude's aggression would have killed.

✓ Cumulative PnL is Consistently Positive

The equity curve shows steady upward progression: +14R at trade 25, +46R at 100, +68R at 150, +104R at 249. No significant drawdown period. The system hasn't had a losing 25-trade stretch. However, the rate of gain is slowing: the first 100 trades averaged 0.46R, the last 49 averaged 0.26R.

⚠ AutoGluon Is Not Functioning

The classifier was trained once on 18 March (92 samples, 77.2% accuracy) but the model files are empty — the trained models aren't persisted to disk or were lost. Despite reporting "autogluon_win_prob" in every calibration JSON, the value is 0.000 across all 249 trades. AutoGluon is contributing nothing to calibration. The quantile regressor was never trained (needs 50+ samples, but only 32 live trades exist in the dataset).

⚠ Confidence Grades Are All D and F

Every single trade has been graded D (76%) or F (24%). Average confidence score: 0.20 (on a 0-1 scale). This is because: (1) AutoGluon contributes 0.0 to the score, (2) historical matching finds "unknown" setup types, and (3) all trades are classified as "unknown" setup type. The grading system can't work properly without functioning AutoGluon and proper setup classification.

✗ Session Tagging Is Broken

The accuracy tracker records ALL 389 trades as session "off" — even though the scanner correctly identifies killzones (Asian, London, NY AM, etc.) in the setup data. The bridge between scanner → accuracy tracker is not mapping killzone strings to session categories. This means the system can't learn per-session performance, which was a core design goal.

✗ Setup Type Classification Is Broken

All 389 trades are classified as setup_type "unknown". The claude_bridge's setup classifier (which should tag as "bull_ob_fvg_sweep_london" etc.) isn't being called or isn't receiving the necessary ICT element data from Claude's analysis. Without proper setup types, historical pattern matching can't find meaningful similar trades, and per-setup-type learning is disabled.

⚠ Data Inconsistency: 400 Bayesian Trades vs 249 Scanner Trades

The Bayesian state shows 400 total trades (312 wins, 88 losses) but the scanner only has 249 resolved trades (185 wins, 64 losses). The extra ~151 trades likely came from manual /trade/complete calls or the V1 seeding process. This isn't necessarily wrong (the Bayesian state should accumulate all signals) but it means the Bayesian WR (76.9%) reflects a different population than the scanner WR (74.3%).

⚠ Live Trade Ingestion to Dataset Is Stalled

The training dataset has 92 rows: 60 from WFO seeding + 32 live trades. But the scanner has resolved 249 trades. The remaining 217 resolved trades aren't being ingested into the training CSV. This means AutoGluon retraining (even if the models were persisted) would be training on stale, incomplete data.

⚠ Performance Is Decaying Slightly

Rolling 50-trade win rate: 80% → 68% → 78% → 76% → 69%. Average RR per 50 trades: 0.55R → 0.37R → 0.44R → 0.47R → 0.26R. The last batch (trades 201-249) shows both the lowest WR and lowest avg RR. This could be noise, but it's worth monitoring — the system should be getting better over time, not worse.

Verdict: Is the Model Improving Itself Over Time?

Partially. The Bayesian layer is genuinely learning — it started with pessimistic V1 priors and has correctly converged to Claude's actual ~77% win rate. SL calibration is consistently adding value by saving ~13% of trades from premature stops.

However, the system is not fully self-improving as designed. AutoGluon (the adaptive ML component) is dead — trained once but models weren't saved. Setup classification and session tagging are broken, so the system can't learn which specific patterns and sessions work best. Live trades aren't flowing into the training dataset, so even fixing AutoGluon would train on stale data. The confidence grading system is non-functional (all D/F grades).

What's working: Claude's analysis (74% WR, +104R), Bayesian belief tracking, SL widening.
What's broken: AutoGluon persistence, setup type classification, session mapping in accuracy tracker, live trade → dataset ingestion pipeline.

The system is profitable, but it's running on only 2 of its 6 calibration layers. Fixing the data pipeline issues would unlock the full adaptive loop.
