Standard Scan Path — Every 15min to 1day

This is the main flow for every scan cycle. Follow the numbered steps. Colour = who's making the decision. Orange nodes are the Narrative Engine maintaining state between scans. Purple is Opus thinking strategically. Blue is Sonnet doing the trade analysis. Pink is ML adjusting the numbers.

1. Scheduler Fires

Timer triggers scan_all_timeframes() in scanner.py. Checks which timeframes have new candle data via hash comparison. Only proceeds if candles actually changed.

Candle counts: 15min→96, 1H→120, 4H→40, 1D→40

2. Narrative Engine — Fetch Previous Thesis

NarrativeStore.get_current(timeframe) retrieves the last thesis Sonnet produced for this timeframe. Runs check_invalidation() against current price — has price blown through the invalidation level?

Structural Contradiction Check: Computes compute_market_structure() score. If thesis says BULLISH but structure scores bearish (<-0.2), injects a contradiction warning into the prompt so Sonnet is forced to re-evaluate rather than anchoring on its old thesis.

State persists between scans. This is the memory of what Claude thought last time.

3. Haiku Pre-Screen — "Is there anything here?"

Decision: PASS (continue) or REJECT (log rejection for FN tracking, exit)

7. ★ SONNET ANALYSIS — The Core Decision

_call_claude() sends everything to Sonnet in one mega-prompt built by build_enhanced_ict_prompt():

Inputs Sonnet receives:
• Execution-TF candles (OHLCV) + HTF candles
• Previous thesis from Narrative Engine (with invalidation status + structural contradiction warnings)
• Opus macro narrative (bias, P3 phase, key levels)
• Intermarket context (DXY correlation, yield direction)
• Recent context (what resolved recently, consumed zones)
• Learned rules from setup profiles + entry placement guidance

Sonnet decides:
• Direction (long/short) + entry price
• Stop loss + take profits (SL/TP1/TP2/TP3)
• Setup quality grade (A/B/C/D)

10. ML Calibration — "Adjust the numbers"

_calibrate() → ClaudeAnalysisBridge.parse_analysis() → MLCalibrator.calibrate_trade()

Runs Sonnet's entry/SL/TP through 6 layers:

Layer 1: Volatility Calibrator — ATR + session scaling + regime
Layer 2: V1 Session Stats — baseline drawdown distributions from 1151 simulated trades
Layer 3: Bayesian Beliefs — learned win rate (Beta distribution, alpha/beta from live trades)
Layer 4: AutoGluon Quantile — ML model predicts optimal SL/TP bands (if trained)
Layer 5: Historical Pattern Matching — similar setups by volatility profile
Layer 6: Consensus — widest SL, median TPs across all layers

Output: Calibrated SL/TP (usually wider SL than Sonnet gave, to survive gold volatility)

Narrative Engine — The System's Memory

Without the narrative engine, every scan would be a cold reading — Claude sees candles with no context of what it thought 15 minutes ago. The narrative engine gives Claude continuity: "Last scan you said we're in accumulation with a bullish bias. Price has since dropped through your invalidation level. Re-evaluate."

What It Stores (per timeframe)

thesis — free text narrative (e.g., "Accumulation phase forming with liquidity sweep below 2340")

directional_bias — bullish / bearish / neutral

bias_confidence — 0.0 to 1.0

p3_phase — accumulation / manipulation / distribution

p3_progress — early / mid / late

key_levels — JSON array of price levels to watch

expected_next_move — what Claude thinks happens next

invalidation — price level that kills the thesis

watching_for — specific triggers to confirm thesis

scan_count — how many scans this thesis has survived

is_revision — did bias change from previous?

Anti-Anchoring Safeguards

1. Stale thesis expiry
15min: 2 hours, 1H: 8 hours, 4H: 24 hours, 1D: 72 hours. Old theses auto-expire so Claude doesn't anchor on ancient analysis.

2. Invalidation check
check_invalidation() runs BEFORE the prompt is built. If price blew through the invalidation level, Sonnet is told explicitly.

3. Structural contradiction
compute_market_structure() compares Claude's thesis direction against actual structure score. If they conflict, a warning is injected.

4. Confidence decay
If expected_next_move doesn't happen, confidence decays by 15% per scan. Prevents stale-but-confident theses from persisting.

5. Revision detection
When bias flips, it's flagged as is_revision=True. If a pending setup exists on this TF, Stage 6 THESIS_REVISED notification fires.

Data Flow: How the Narrative Feeds Into Everything

→ Into Sonnet's prompt (step 7): Previous thesis + invalidation status + structural contradiction warning. This is how Sonnet "remembers" its last analysis.

→ Into feature extraction: thesis_confidence, p3_progress_encoded, thesis_scan_count, opus_sonnet_agreement become 4 of the 52 ML features. AutoGluon can learn that "thesis survived 3+ scans" correlates with winning trades.

→ Into lifecycle notifications: Stages 1/2/6 use narrative state to notify you when a thesis is forming, confirmed, or revised.

→ Into the prospect path: Opus uses the active thesis to decide where to pre-position zone alerts before killzones.

ML Calibration — How the Numbers Get Adjusted

Sonnet makes the trade decision (direction, entry, SL, TP). The ML stack doesn't decide whether to trade — it adjusts where the SL and TPs should be, based on statistical evidence. The final SL is always the widest across all layers (protects you), and TPs are the median.

LayerWhat It DoesInputsWhen ActiveModule
1. Volatility Calibrator Scales SL/TP based on current ATR, trading session, and 5-state volatility regime (low/normal/elevated/high/extreme) Candles → ATR, session (Asian/London/NY), regime classification Always (from first trade) volatility.py
2. V1 Session Stats Baseline drawdown distributions from 1151 V1 simulated trades. "At this session, 80th percentile drawdown is X ATR." Pre-computed session statistics (seeded once) Always (if seeded) calibrate.py + seed.py
3. Bayesian Beliefs Sequential learning: Beta(alpha, beta) distribution for win rate. Updated after every live + scanner trade. Prior: alpha=8.92, beta=11.08 (44.6% WR from V1) Prior state + each trade outcome Always bayesian.py
4. AutoGluon Quantile ML model predicts optimal SL/TP bands from 52 features. Binary classifier (win/loss) or Multi-3 (stopped/tp1/runner) depending on data quality. 52-column feature vector from features.py After 30+ trades (classifier) or 100+ rich-feature trades (multi-3) prediction.py + training.py
5. Historical Matching Finds past setups with similar volatility profile. Uses their actual outcomes to suggest SL/TP. Current ATR, session, regime → match against training dataset When training dataset has enough data calibrate.py
6. Consensus Takes the widest SL and median TPs across all active layers. Computes composite confidence + final grade. All layer outputs Always consensus.py

How AutoGluon Gets Trained

1. Every resolved trade → dataset.ingest_live_trade() adds a row with 52 features + outcome

2. should_retrain() checks every N trades or every 6 hours

3. train_classifier() loads blended dataset (V1 seed data + live trades, weighted by feature quality)

4. Binary mode (win/loss) until 100+ rich-feature rows, then auto-upgrades to Multi-3 (stopped/tp1/runner) if OOS accuracy > 0.45

5. Quantile regressor trained separately for SL/TP band predictions

6. Model saved to disk, prediction.py auto-detects which type is active

Where Backtest Engine Fits

BacktestGenerator + WalkForwardEngine exist to generate training data and validate the ML models. They are NOT in the live scan path.

Flow:
1. BacktestGenerator fetches historical candles from OANDA
2. Runs structural scan to find potential setup zones
3. Calls Haiku/Sonnet to evaluate each zone
4. Labels outcomes from forward price action
5. WFO runs walk-forward cross-validation
6. Results feed into dataset.ingest_wfo_trades()

Purpose: Generate enough training data before live data accumulates. V1 seed data (1151 trades) bootstraps the system.

The Flywheel Effect

More trades resolve → more data for every feedback loop → AutoGluon gets sharper, Bayesian beliefs tighten, DNA patterns accumulate, Haiku false negatives are learned, Opus accuracy is tracked, narrative field weights adjust, prompt variants are tested...

Every feedback loop is independent and degrades gracefully — if any one fails, the others keep working. But together, they create a system that gets measurably better with every trade.
